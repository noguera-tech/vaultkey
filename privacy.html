Política de Privacidad

Última actualización: Junio 2026

Introducción

VaultKey es una aplicación de bóveda digital privada diseñada para permitir al usuario guardar contraseñas, notas, tarjetas, documentos, licencias, datos médicos, credenciales WiFi y otra información sensible de forma local y cifrada.

La privacidad y la seguridad son principios fundamentales de VaultKey. Esta política explica qué datos puede introducir el usuario, cómo se almacenan, qué funciones opcionales pueden usar servicios externos y cómo puede contactar con el desarrollador.

Información que el usuario puede guardar

VaultKey no crea cuentas de usuario en servidores propios ni solicita datos personales obligatorios para usar la bóveda local. Sin embargo, el usuario puede guardar voluntariamente información privada dentro de la aplicación, incluyendo:

  • Contraseñas, usuarios, correos electrónicos y sitios web.
  • Notas privadas.
  • Datos de tarjetas u otra información financiera introducida voluntariamente.
  • Documentos, licencias, datos WiFi y otros registros personales.
  • Información médica u otros datos sensibles que el usuario decida guardar.
  • Datos importados desde archivos CSV compatibles, cuando el usuario utiliza la función de importación.

El usuario no debe introducir datos reales durante pruebas públicas o cerradas si no desea hacerlo.

Almacenamiento local y cifrado

La información guardada en VaultKey se almacena localmente en el dispositivo del usuario y se protege mediante cifrado. VaultKey utiliza cifrado AES-GCM y derivación de clave PBKDF2 para proteger la bóveda a partir del PIN o método de acceso configurado.

  • El contenido de la bóveda no se envía a servidores propios de VaultKey.
  • VaultKey no vende los datos del usuario.
  • El usuario es responsable de mantener la confidencialidad de su PIN, dispositivo y respaldos.

PIN, recuperación y pérdida de acceso

El PIN se utiliza para proteger y desbloquear la bóveda. Si el usuario pierde el PIN y no dispone de un método de recuperación válido, es posible que no pueda recuperar el acceso a la información cifrada. VaultKey no puede garantizar la recuperación de datos si técnicamente no existe una recuperación válida configurada.

Sincronización y respaldo opcional con Google Drive

VaultKey puede ofrecer una función opcional de copia de seguridad o sincronización mediante Google Drive. Esta función solo se utiliza si el usuario decide activarla e iniciar sesión con su cuenta de Google.

  • VaultKey puede solicitar autorización de Google únicamente para guardar o leer archivos propios de la aplicación, como respaldos cifrados.
  • El respaldo se almacena cifrado. VaultKey no envía el PIN ni la clave de descifrado a Google Drive.
  • Google Drive actúa como proveedor de almacenamiento externo para el respaldo cifrado.
  • El usuario puede desconectar Google Drive desde la aplicación cuando la función esté disponible.
  • El respaldo puede almacenarse en la carpeta privada de datos de aplicación de Google Drive, conocida como appDataFolder, según la configuración usada por VaultKey.

Respaldos locales e importación

VaultKey puede permitir al usuario exportar e importar respaldos o archivos CSV compatibles. Estas acciones se realizan únicamente bajo acción del usuario.

  • Los respaldos deben conservarse en lugares seguros.
  • Los archivos importados deben pertenecer al usuario.
  • Un archivo incorrecto, dañado o incompatible puede no ser aceptado por la aplicación.

Acceso al dispositivo y funciones utilizadas

VaultKey puede utilizar funciones del dispositivo necesarias para ofrecer seguridad y funcionamiento básico:

  • Almacenamiento local: para guardar la bóveda cifrada y la configuración de la app.
  • Biometría nativa de Android: huella, rostro u otro método compatible para desbloqueo rápido opcional.
  • Selector o acceso a archivos: para importar o exportar respaldos cuando el usuario lo solicita.
  • Conexión a internet: para cargar la PWA/TWA, publicar actualizaciones y usar Google Drive si el usuario activa esa función.

VaultKey no recibe ni almacena plantillas biométricas. La autenticación biométrica es gestionada por el sistema Android del dispositivo.

Protección visual

VaultKey puede usar mecanismos de protección visual, como bloqueo de capturas de pantalla en Android, para reducir el riesgo de exposición accidental de datos sensibles. Estas medidas ayudan a mejorar la privacidad, pero no constituyen una garantía absoluta frente a dispositivos comprometidos, cámaras externas u otros métodos de captura.

Compartición de datos con terceros

VaultKey no vende, alquila ni comparte el contenido privado de la bóveda con terceros. La aplicación puede interactuar con servicios externos únicamente cuando el usuario utiliza funciones opcionales, como Google Drive para respaldos cifrados. En esos casos, el uso del servicio externo también puede estar sujeto a las políticas de privacidad del proveedor correspondiente.

Retención y eliminación de datos

Los datos de la bóveda permanecen en el dispositivo del usuario hasta que el usuario los elimine, borre los datos de la aplicación o desinstale VaultKey. Si el usuario crea respaldos locales o en Google Drive, esos respaldos permanecerán donde el usuario los haya guardado hasta que los elimine manualmente o mediante las opciones disponibles en la aplicación o en el servicio correspondiente.

VaultKey no mantiene cuentas propias de usuario en servidores propios. Para eliminar datos locales, el usuario puede borrar las entradas guardadas, borrar los datos de la aplicación desde Android o desinstalar la app.

Seguridad

VaultKey aplica medidas de seguridad como cifrado local, bloqueo automático, PIN, autenticación biométrica opcional y protección visual en Android. No obstante, ningún sistema puede garantizar una seguridad absoluta. El usuario debe proteger su dispositivo, su PIN, sus respaldos y su cuenta de Google si decide activar Google Drive.

Menores de edad

VaultKey no está dirigida específicamente a menores de 13 años. Si una persona menor de edad utiliza la aplicación, debe hacerlo con la supervisión de sus padres, representantes o tutores legales, según las leyes aplicables en su país.

Cambios en esta política

Esta política puede actualizarse ocasionalmente para reflejar mejoras, cambios en VaultKey, requisitos legales o políticas de Google Play. Las modificaciones se publicarán en esta misma página y se actualizará la fecha indicada al inicio.
